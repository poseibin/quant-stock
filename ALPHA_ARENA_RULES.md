# Alpha 打擂规则

本文档定义通用策略和进攻策略的打擂规则。核心原则：不要把“稳健准入”和“高收益进攻”混在同一个评分里比较。

## 1. 两条策略线

### 稳健 Alpha 线

用途：

- 作为通用策略的稳健底仓。
- 作为其他进攻策略的风控过滤器。
- 追求低回撤、低换手、压力分段可控。

当前擂主：

- 版本：`v53 Alpha动态暴露 overheat045 cw020`
- 模型：`fr_smallcap_attack_raw_20100101_20260612_20260613_043715`
- 评估：`alpha_regime_active_v53_20260614`
- 准入分：`72.83`
- 年化收益：`4.51%`
- 最大回撤：`-6.08%`
- Sharpe：`1.239`

稳健线的目标区间：

- 年化收益：`8% - 15%`
- 最大回撤：`8% - 15%`
- Sharpe：尽量 `> 1.2`
- 压力分段不能明显失效。

### 进攻 Alpha 线

用途：

- 追求高收益，不与稳健线直接按准入分比较。
- 只在小盘生态强、赚钱效应强、涨停扩散强、风格偏小盘时启用。
- 平时可空仓或由策略路由降低权重。

进攻线的第一阶段目标：

- 强势 regime 下年化：`30% - 50%`
- 最大回撤暂允许：`25% - 35%`
- 后续再通过 regime、meta labeling、仓位模型压回撤。

进攻线不能直接覆盖稳健线。只有在明确通过进攻评估后，才能作为单独策略加入策略路由。

## 2. 为什么 72 分不等于高收益

当前准入分是综合分，不是收益分。评分权重来自 `quant_stock_core/scripts/evaluate_strategies.py`：

| 组件 | 权重 | 含义 |
|---|---:|---|
| `return_score` | `30%` | 年化收益得分 |
| `drawdown_score` | `5%` | 最大回撤 |
| `risk_adjusted_score` | `15%` | Sharpe / Calmar |
| `cost_score` | `12%` | 换手成本 |
| `capacity_score` | `6%` | 成交额和市值容量 |
| `stability_score` | `20%` | 月胜率、3 个月正收益率、最差月 |
| `independence_score` | `12%` | 与基准相关性和持仓重合度 |

v53 年化只有 `4.51%`，但准入分能到 `72.83`，主要是因为：

- 最大回撤只有 `-6.08%`，回撤分为 `100`。
- 换手很低，成本分为 `100`。
- 与基准重合度低，独立性分为 `100`。
- 压力测试没有被扣分。

所以稳健线准入分高，只代表“这个策略适合放进稳健候选池”，不代表它是高收益进攻策略。

## 3. 稳健线打擂规则

### 3.1 比较口径

稳健线按 `admission_score` 排名，但必须同时看：

- `annual_return`
- `max_drawdown`
- `sharpe`
- `stress_crash_state_failed`
- `stress_weak_drawdown_failed`
- `reason`
- `avg_holdings`
- `avg_turnover`

### 3.2 可激活条件

新版本只有满足以下条件，才能覆盖 active：

- `admission_score` 明确高于当前 active。
- `annual_return` 不低于当前 active，或者收益略低但回撤/Sharpe 明显更优。
- `max_drawdown` 不明显恶化。
- `stress_bad_event_count = 0`。
- 不能出现：
  - `急跌状态收益显著失效`
  - `弱市回撤`
  - `裸策略回撤超过实盘线`
  - 关键压力段失效。
- 必须复验一次，保存独立评估 run。
- 复验通过后，才写入 `strategy_config_versions` 并更新 `strategy_model_active`。

### 3.3 不可激活但可观察

以下情况只能标记为“继续观察”：

- 年化提升但急跌/弱市失败。
- 回撤下降但收益太低。
- 平均持仓太少导致容量弱。
- 仅单一指标提升，综合收益/回撤没有改善。

### 3.4 v53 这轮经验

v52 到 v53 的有效变化：

- `filters.index_anchor_warning.overheat_exposure`: `0.40 -> 0.45`
- `filters.crash_warning_model.overheat_exposure`: `0.04 -> 0.02`

结果：

- 准入分：`72.75 -> 72.83`
- 年化：`4.48% -> 4.51%`
- 回撤：`-6.09% -> -6.08%`
- Sharpe：`1.232 -> 1.239`

这说明 v52/v53 附近已经是一个窄峰。继续微调过热/预警暴露，大概率只能拿到很小收益，不适合期待年化跃迁。

## 4. 进攻线打擂规则

### 4.1 进攻线不能使用稳健准入分作为唯一目标

进攻线应该单独排序，建议临时排序指标：

```text
attack_score =
  annual_return * 100
  - max(0, abs(max_drawdown) - 0.20) * 80
  + sharpe * 2
```

这个分数只用于研究排序，不用于实盘激活。

### 4.2 进攻线核心指标

必须单独展示：

- 强势 regime 年化。
- 全周期年化。
- 最大回撤。
- 强势 regime 最大回撤。
- Top10 / Top20 未来 `5/10/20` 日净收益。
- 胜率。
- 单票平均收益。
- 换手和成本后收益。
- 急跌、弱市、流动性挤压状态下是否自动关闭。

### 4.3 进攻线候选条件

第一阶段候选不要求低回撤，但要满足：

- 强势 regime 年化接近或超过 `30%`。
- 全周期不能靠单一年份贡献。
- 最大回撤暂不超过 `35%`。
- 能明确解释什么时候开仓、什么时候关闭。
- 不能依赖未来数据或盘后不可用数据做盘中判断。

### 4.4 年化 50% 的现实判断

当前通用因子模型做不到年化 `50%`。已验证：

- 放开弱市暴露、急跌保护、候选池、持仓数后，年化大致只能到 `6% - 9%`。
- 同时最大回撤快速扩大到 `20% - 30%`。

这说明瓶颈不是风控太严，而是通用因子模型本身的 alpha 强度不够。

如果目标是年化 `50%`，必须新建进攻策略线，而不是继续微调 `ml_factor_ranker`。

## 5. 下一步研发路线

### 5.1 保留 v53

v53 作为稳健版继续保留：

- 不用它追求年化 `50%`。
- 用它做小盘质量过滤、风险底座、组合稳健袖珍仓位。

### 5.2 新建小盘强势进攻模型

建议新策略名：

- `smallcap_attack`
- `smallcap_momentum_attack`
- `smallcap_ecology_attack`

第一版目标：

- 只在强势小盘生态中出手。
- 训练未来 `5/10/20` 日收益排名。
- 更重视涨停扩散、昨日涨停表现、连板高度、赚钱效应、成交额扩张、小盘相对强弱。
- 允许更高换手和更集中持仓。

### 5.3 策略路由

最终不是让一个模型永远运行，而是：

```text
市场状态 / 小盘生态
  -> 稳健 v53 / 小盘进攻 / 涨停 / 横盘 / 做T / 空仓
  -> 动态仓位
  -> 风控退出
```

路由规则：

- 小盘生态强：允许进攻策略满仓或高仓位。
- 小盘生态弱：只保留 v53 小仓位或空仓。
- 急跌/流动性挤压：关闭进攻策略。
- 修复期：逐步恢复。

## 7. 激活纪律

- 稳健线和进攻线分开激活。
- 稳健线可以覆盖 `ml_factor_ranker` active。
- 进攻线不能覆盖 `ml_factor_ranker` active，必须单独策略或单独版本路线。
- 任何激活必须有：
  - 独立复验 run。
  - active 模式复验 run。
  - active 指针查询确认。
  - 至少相关 Python 单测通过。
